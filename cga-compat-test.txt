Batch Mode
----------
CGA_COMP supports command-line batch mode for unattended test plates:
  CGA_COMP /nXX rbars cbars aspect motion audiosync
  /nXX = delay XX seconds per plate (default 5)

Interactive Mode
----------------
Arrow keys + ENTER to navigate menus.
Menu structure:
  - Information
  - CGA Compatibility        > (submenu)
  - RGBI Compatibility       > (submenu)
  - Monitor Calibration Plates > (submenu)
  - Custom Calibration Plates  > (submenu)
  - Color CGA Speed Benchmark > (submenu)
  - About This Program

Test Features (from docs):
  - Adapter memory speed benchmarks (read/write, interleaved/dedicated)
  - Color Select / Mode Control register tests (border, background, palettes)
  - Textmode manipulation (40-col, highcolor background, cursor, snow)
  - Monitor calibration (brightness, contrast, moire, 16 colors)
  - MC6845 CRTC programming (hretrace, vretrace, row reprogram 80x100,
    row/col reprogram 90x30, interlaced, sync position, start address)

ludica-mcp Setup
----------------
The TCP daemon is ludica-launcher (NOT ludica-mcp). Start it BEFORE
entering Claude Code so the MCP bridge tools load on session init:

  LUDICA_MCP_ALLOWEXEC=$(echo _out/*/bin/* | tr ' ' ':') \
    _out/x86_64-linux-gnu/bin/ludica-launcher --port=4000 &

The MCP bridge (ludica-mcp-bridge) is configured in .mcp.json
and connects to the launcher on TCP 4000. Once loaded, tools appear as
mcp__ludica__spawn, mcp__ludica__screenshot, mcp__ludica__action, etc.

Known issue: if the launcher is not running when the session starts, the
bridge fails to connect and Claude Code does not retry. The tools will be
absent for the entire session. Fix: kill the session and restart with the
launcher already running.

Workflow for interactive testing:
  1. Use /ludica-mcp skill
  2. spawn(alias="lilpc", args=["--bios=samples/lilpc/bios/pcxtbios.bin",
     "--fda=/tmp/cgacompat/testdisk.img"])
  3. screenshot() to observe CGA_COMP menus
  4. action() to inject keys for navigation (arrow keys, Enter, Escape)
  5. Step through each submenu, screenshot results, record PASS/FAIL

The test disk is at /tmp/cgacompat/testdisk.img (may not survive reboot).
To recreate:
  mkdir -p /tmp/cgacompat
  cp samples/lilpc/disk/demodisk.img /tmp/cgacompat/testdisk.img
  unzip -o /home/jon/Source/CGACompatibilityTester/CGA_COMP.ZIP CGA_COMP.EXE -d /tmp/cgacompat/
  mcopy -o -i /tmp/cgacompat/testdisk.img /tmp/cgacompat/CGA_COMP.EXE ::
  printf "@ECHO OFF\r\nCGA_COMP\r\n" | mcopy -o -i /tmp/cgacompat/testdisk.img - ::AUTOEXEC.BAT
