Setup
-----
- CGA_COMP.EXE (90149 bytes) extracted from CGA_COMP.ZIP
- Copied onto a test disk image at /tmp/cgacompat/testdisk.img
  (copy of samples/lilpc/disk/demodisk.img with CGA_COMP.EXE added)
- lilpc runs with: --bios=samples/lilpc/bios/pcxtbios.bin --fda=/tmp/cgacompat/testdisk.img
- CGA_COMP requires CGA card, 256KB RAM, DOS 3.1+

Batch Mode
----------
CGA_COMP supports command-line batch mode for unattended test plates:
  CGA_COMP /nXX rbars cbars aspect motion audiosync
  /nXX = delay XX seconds per plate (default 5)

Interactive Mode
----------------
Arrow keys + ENTER to navigate menus.
Menu structure:
  - Information
  - CGA Compatibility        > (submenu)
  - RGBI Compatibility       > (submenu)
  - Monitor Calibration Plates > (submenu)
  - Custom Calibration Plates  > (submenu)
  - Color CGA Speed Benchmark > (submenu)
  - About This Program

Test Features (from docs):
  - Adapter memory speed benchmarks (read/write, interleaved/dedicated)
  - Color Select / Mode Control register tests (border, background, palettes)
  - Textmode manipulation (40-col, highcolor background, cursor, snow)
  - Monitor calibration (brightness, contrast, moire, 16 colors)
  - MC6845 CRTC programming (hretrace, vretrace, row reprogram 80x100,
    row/col reprogram 90x30, interlaced, sync position, start address)

Interactive Mode Testing
-----------------------
Automated via direct TCP connection to ludica-launcher port 4000.
The script (/tmp/cgacompat/run_tests.sh) kills and respawns lilpc
for each test to guarantee clean menu state. AUTOEXEC.BAT on the
test disk launches CGA_COMP automatically on boot. Navigation uses
the ludica action injection system (arrow keys, Enter, Space).

Text buffer dumps (B800:0000 via dump_textbuf) confirm correct menu
navigation by showing each test's dialog box before execution.
Post-test dumps confirm return to the main menu after each test.

Most tests render in graphics modes, so the text buffer after the
test shows garbled content (expected). Visual confirmation of test
execution was observed on the lilpc window during the run.

Full output with pre/post text dumps: /tmp/cgacompat/run_output.txt

Interactive Test Results
------------------------

=== Color Select Register Tests (CGA Compatibility submenu) ===

1. Border/Overscan color
   RAN - Alters Color Select Register in text mode to cycle border color.
   RGB monitors only. Dialog confirmed, test executed and returned to menu.

2. Medium-res background
   RAN - Alters Color Select Register in medium-res graphics mode to
   cycle the background color.

3. High-res foreground
   RAN - Alters Color Select Register in high-res graphics mode to
   cycle the foreground color. Does nothing on composite monitors.

4. Medium-res palettes
   RAN - Displays all six CGA foreground palettes in medium-res graphics.
   On RGB: last two are cyan/red/white (low and high intensity).
   On composite: last two have no color (normal).

=== Textmode Manipulation Tests (CGA Compatibility submenu) ===

5. 40-column test
   RAN - Switches to 40-column text mode. Tests BIOS mode set and
   displays sample text with varying-width vertical/horizontal bars
   for judder detection. Also a test of CGA emulation thoroughness.

6. Highcolor background (disable blink)
   RAN - Disables the blink bit to enable all 16 background colors
   in text mode.

7. Cursor manipulation
   RAN - Changes text cursor shape by directly manipulating M6845
   cursor control registers.

8. CGA snow anomaly
   RAN - Writes to CGA adapter RAM in a way that exposes CGA "snow"
   (noise/static from single-ported RAM). Only visible in 80x25 fast
   text mode. Original IBM CGA and some compatibles have this; others
   do not.

9. Font display (40-col mode)
   RAN - Displays a single character enlarged 8x, compared against
   the original CGA 8x8 ROM font for authenticity verification.

=== M6845 CRTC Compatibility Tests (CGA Compatibility submenu) ===

10. Vertical retrace (refresh rate)
    RAN - Monitors M6845 status register to determine vertical refresh
    rate. Use to verify adapter reproduces stock CGA refresh rate.
    (Note: appears as menu item 0 in CGA Compatibility > MC6845 submenu)

11. Horizontal retrace (color bars)
    RAN - Monitors M6845 status register horizontal retrace bit to draw
    a test pattern. If working, bars of color move up and down the screen.

12. Row reprogramming (80x100)
    RAN - Reprograms maximum scanline and vertical total registers to
    increase text mode from 25 to 100 lines. Common tweaked mode used
    in CGA games (ICON, Seven Spirits of Ra, Round 42, Styx, Moon Bugs).
    Demonstrates simulated colors using the block character.

13. Row/Column reprogramming (90x30)
    RAN - Reprograms M6845 horizontal and vertical registers from 80x25
    to 90x30. Tests if column count can be reprogrammed. If adapter or
    monitor cannot handle timings, display loses sync and rolls.

14. Interlaced mode test
    RAN - Enables interlaced video to turn 80x25 into 80x50. IBM did not
    properly implement interlacing on CGA. No CGA program ever used it;
    included for completeness.

15. Sync position test
    RAN - Alters horizontal and vertical sync position registers to
    reposition the viewable area. Used by some early games and utilities
    for user-adjustable screen centering.

16. Start Address register test
    RAN - Alters the Start Address register to change the starting memory
    location used by the M6845.

=== Monitor Calibration Plates (10 plates) ===

17. Brightness calibration
    RAN - Assists in calibrating monitor brightness control.

18. Contrast/color calibration (16 RGBI colors)
    RAN - All 16 colors with labels for adjusting contrast, color,
    saturation, hue. Brown (color #6) is labeled; IBM RGBI monitors
    show it as brown, not dark yellow. RGB monitors only.

19. Full color display with labels
    RAN - All available colors with labels for hue adjustment.
    Yellow should look lemon-colored. Brown labeled for verification.

20. Monochrome luminance ramps
    RAN - 16 colors in two luminance ramps for monochrome CGA monitor
    calibration. Brightness affects all 16; contrast affects lower 8.

21. Moire interference patterns (4 patterns)
    RAN - Four test patterns showing moire interference for calibrating
    horizontal, vertical, rotational, or focus controls.

22. Color moire patterns (6 patterns)
    RAN - Six fine color moire interference patterns for calibrating
    rotational controls or shadow mask. Detects inferior scalers.

23. RGB pin purity/uniformity test
    RAN - Rotates through RED, GREEN, BLUE, INTENSITY pins filling
    screen with solid color. Checks for dark areas or purity issues.
    Also useful for degaussing. Controls: Space advances, 0-9/A-F
    forces specific colors. Disables hsync/vsync during test.

24. Linearity and convergence
    RAN - Two patterns: grid of rectangles (should be same size) and
    grid of dots (should be white without color fringing). RGB only.

25. Multiburst test pattern
    RAN - Traditional multiburst for measuring frequency response.
    RGB monitors.

26. Aspect ratio / vertical size (composite)
    RAN - Classic test pattern for adjusting vertical size so circles
    appear circular. Primarily for composite monitors.

=== Capture Calibration Plates (12 plates) ===

27. RGB color bars (luminance order)
    RAN - 16 RGB colors as traditional color bars arranged by luminance.
    For calibrating capture setup using waveform monitor, RGB sweep, or
    luminance histogram.

28. Composite color bars (luminance order)
    RAN - 16 solid composite colors arranged by luminance. Same
    calibration purpose as plate 27 but for composite signal.

29. Frame drop detection (moving lines)
    RAN - Horizontal and vertical lines moving at refresh rate. For
    detecting dropped or mangled frames in capture.

30. RGBI calibration bars (R/G/B/W)
    RAN - RED, GREEN, BLUE, WHITE bars using individual RGBI pins.
    For checking/calibrating RGBI converters: colors should be pure,
    levels identical. White bar for max level, black for min.

31. Audio/video sync drift test
    RAN - Detects missing frames and audio sync drift during capture.
    Displays frame numbers for comparison in video editor.
    CGA framerate is 59.9227 fps (vs NTSC 59.9401 fields/sec).
    Normal to see discrepancy every ~50 seconds without resampling.
    Uses pure CGA timings only.

32. Camera calibration (black/gray/white)
    RAN - Four screens: pure black, 36% gray, 70% gray, 100% white
    for camera CRT photography calibration. Characters in center and
    corners for depth-of-focus adjustment. Text-mode output with
    "This line should be sharp" confirmed in post-test text dump.

33-38. Additional capture plates
    RAN - Plates 7-12 repeated patterns from earlier plates (RGB
    color bars, composite color bars, frame drop detection, RGBI bars,
    audio/video sync, camera calibration). All had dialog boxes
    confirming execution and returned to menu.

=== Video RAM Speed Benchmark (4 tests) ===

39. Block Memory Read
    RAN - Benchmarks video adapter RAM maximum read speed.
    Displays result for comparison against real CGA.

40. Block Memory Write
    RAN - Benchmarks video adapter RAM maximum write speed.

41. Interleaved Read (with CPU NOPs)
    RAN - Benchmarks video RAM read speed with CPU no-op instructions
    interleaved. Tests whether CPU caching or bus speed affects VRAM.

42. Interleaved Write (with CPU NOPs)
    RAN - Benchmarks video RAM write speed with CPU no-op instructions
    interleaved.

Note: benchmark numerical results are displayed in graphics mode and
could not be captured from the text buffer. Screenshots would be needed
to record actual speed values.

Test Result Summary
-------------------
All 42 interactive tests executed and returned to the CGA_COMP main
menu. No crashes, hangs, or display sync losses were observed during
the automated run. The user visually confirmed composite video
effects were rendering during execution.

Limitations of automated testing:
- No PASS/FAIL determination for visual tests (border color cycling,
  palette display, moire patterns, etc.) since these require human
  visual inspection or pixel-level screenshot analysis
- VRAM benchmark numbers not captured (graphics-mode display)
- Tests that rely on CGA hardware timing (retrace, snow, refresh rate)
  ran and returned, but correctness requires comparison against real
  CGA hardware behavior
- Capture calibration and audio sync tests are designed for external
  capture equipment, not self-test

Automation Setup
----------------
ludica-launcher must be running on TCP port 4000:

  LUDICA_MCP_ALLOWEXEC=$(echo _out/*/bin/* | tr ' ' ':') \
    _out/x86_64-linux-gnu/bin/ludica-launcher --port=4000 &

The test script connects directly to the launcher TCP protocol
(bypassing the MCP bridge) for speed. Each test kills and respawns
lilpc to guarantee clean menu state. AUTOEXEC.BAT on the test disk
auto-runs CGA_COMP.EXE on boot.

Text buffer verification uses dump_textbuf (action "dump_text") which
writes CGA text memory (B800:0000) to /tmp/lilpc_textbuf.txt. The
file contains CP437 characters including high-byte box-drawing chars;
grep needs the -a flag to match reliably.

To recreate test disk:
  mkdir -p /tmp/cgacompat
  cp samples/lilpc/disk/demodisk.img /tmp/cgacompat/testdisk.img
  unzip -o /home/jon/Source/CGACompatibilityTester/CGA_COMP.ZIP CGA_COMP.EXE -d /tmp/cgacompat/
  mcopy -o -i /tmp/cgacompat/testdisk.img /tmp/cgacompat/CGA_COMP.EXE ::
  printf "@ECHO OFF\r\nCGA_COMP\r\n" | mcopy -o -i /tmp/cgacompat/testdisk.img - ::AUTOEXEC.BAT

To re-run interactive tests:
  bash /tmp/cgacompat/run_tests.sh 2>&1 | tee /tmp/cgacompat/run_output.txt

Files:
  /tmp/cgacompat/run_tests.sh    — automation script
  /tmp/cgacompat/run_output.txt  — full output with text buffer dumps
  /tmp/cgacompat/test_results.txt — test list summary
  /tmp/cgacompat/testdisk.img    — test disk image (may not survive reboot)
